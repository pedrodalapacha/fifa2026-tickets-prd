## Story

**Como** aluno do workshop "Living Lab Azure-Native",
**eu quero** completar a Fase 2 em 6h implementando um gateway profissional em código C# com YARP hospedado em Container App, colocando-o à frente das Functions de F1 e configurando preocupações transversais (rate limiting, output cache, CORS, header transform, path rewrite) em código .NET legível e inspecionável,
**para que** eu compreenda o que um API Gateway faz por dentro — como rate-limit, cache, transformação de headers e validação de JWT são mecanismos de código, não caixas-pretas gerenciadas — e tenha o gateway versionado no repo como ponto de entrada único do fluxo v2, base para F3 (identidade/JWT) e F6 (Flow Visualizer).

## Dev Notes

### Objetivos pedagógicos (6 níveis — conforme Orion/ADE-004)

| Nível | Conceito | O que o aluno aprende |
|---|---|---|
| **0 — Ponto único de entrada** | Reverse proxy, padrão API Gateway/BFF | Cliente nunca vê URL real da Function; desacoplamento de topologia |
| **1 — Roteamento** | Path/host-based routing | `routes` + `clusters` do YARP em `appsettings.json` |
| **2 — Preocupações transversais (coração)** | Rate limiting, auth JWT, CORS, header transform, path rewrite | `AddRateLimiter` (→ 429), `AddJwtBearer` (placeholder F3), `AddCors`, transforms `X-Correlation-ID` — tudo em C# legível |
| **3 — Resiliência** | Load balancing, health checks, timeouts, retries | Clusters YARP com múltiplas destinations; health probes ativo/passivo |
| **4 — Observabilidade de borda** | Logging/tracing centralizado | Gateway é o **primeiro nó** do Flow Visualizer (F6); `X-Correlation-ID` propagado desde aqui |
| **5 — Meta-aprendizado** | Trade-off "comprar gerenciado vs construir" | Custo, controle, provisioning: APIM (US$50-80, 30-45min) vs YARP (US$0, segundos); "em produção corporativa, o equivalente gerenciado é o APIM" |

### Arquitetura técnica delta

```
[Browser]
   │ POST /purchase  (URL do Container App do aluno)
   ▼
[Container App: gateway-<iniciais>  (ASP.NET Core + YARP .NET 8)]
   ├─ UseCors("frontend")
   ├─ UseRateLimiter  ── 429 se > 5/min
   ├─ UseAuthentication / UseAuthorization  ── placeholder F3 (anônimo em F2)
   ├─ RequestTransform: X-Correlation-ID injetado  ── ADE-000 Inv 5
   ├─ OutputCache  ── cache 30s em GET
   └─ MapReverseProxy ── roteia para Function App F1 do aluno
            │
            ▼
     [Function App F1: PurchaseEntryFunction]
            └─ (resto do fluxo F1: Service Bus → Consumer → SQL)
```

> **NÃO toca:** Node API (`fifa2026-api/`), frontend Vite, SQL schema, Functions F1 (apenas recebem requisições via gateway agora). A connection string `SqlConnectionString` permanece nas Functions, não no gateway (conforme ADE-003 Invariante 3).

| `cache-lookup` / `cache-store` | `AddOutputCache` (30s, header X-Cache HIT) | AC-6 |

### Troubleshooting esperado

| Sintoma | Causa | Mitigação |
|---|---|---|
| 502 Bad Gateway do Container App | URL da Function errada em `FunctionAppF1Url` | Verificar App Setting no Container App; confirmar que Function usa `authLevel: Anonymous` (F1) |
| Rate-limit não dispara (sem 429) | `UseRateLimiter` antes de `MapReverseProxy` mas `RequireRateLimiting` não aplicado à rota | Adicionar `RequireRateLimiting("fixed")` na configuração do proxy |
| X-Correlation-ID não aparece no backend | Transform configurado em `clusters` em vez de `routes` | Mover `RequestTransforms` para a seção de route |
| Output cache não retorna HIT | Policy de cache aplicada ao POST em vez do GET | Segregar routes: uma para POST (sem cache), outra para GET (com cache) |
| Container App cold start alto | Consumption plan scale-to-zero | Aceitar em workshop (didático — mencionar "min replicas: 1" em prod) |
| JWT placeholder bloqueia chamadas em F2 | `RequireAuthorization()` aplicado antes de F3 | Confirmar que rotas estão com `.AllowAnonymous()` em F2 |

### Correlation propagation pattern (preparatório para F6 — ADE-000 Inv 5)

- **Gateway injeta:** `X-Correlation-ID` via YARP `RequestTransform` (novo GUID se ausente)
- **Function recebe:** header `X-Correlation-ID` e propaga via `ApplicationProperties["CorrelationId"]` no Service Bus
- **Gateway é o nó zero** do Flow Visualizer de F6 — o primeiro evento de tracing começa aqui
